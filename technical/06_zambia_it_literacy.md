# スマホの普及がデジタルデバイドを「解消しなかった」メカニズム

技術者として、ある前提を疑ったことがなかった。というより、疑う必要すら感じていなかった。

「インターネットに接続できれば、情報にアクセスできる。情報にアクセスできれば、格差は縮まる。」

我々が当然視しているこのロジックは、技術の世界では暗黙の公理のように扱われている。ザンビアの現地調査で、この公理が根本的に崩れた。

## フィールドノート：AIの認知度ゼロという事実

ザンビアでの水資源調査に従事していた際、現地レストランのスタッフに「AIを知っているか」と尋ねてみた。回答は、ほぼ全員が「知らない」。名前すら聞いたことがないという反応だった。

ITU（国際電気通信連合）の統計によれば、ザンビアのモバイルセルラー契約数は人口100人あたり約100件（2023年時点）に達しており、複数SIM保有を考慮してもモバイル通信の物理的カバレッジは広い。インターネット利用率は人口の約30%（World Bank, 2023）、モバイルブロードバンド契約は人口100人あたり約30件で推移している（ITU, 2023）。ZICTA（Zambia Information and Communications Technology Authority）の報告でも、アクティブなモバイルインターネット加入者数は増加傾向にある。都市部ではスマートフォンの所持は一般的で、Tecno、Itel、Infinixといった中国系ブランドの安価なAndroid端末（50〜150 USD帯）が市場を席巻しており、Samsung Galaxy Aシリーズなども一定のシェアを持つ（GSMA, *The Mobile Economy Sub-Saharan Africa 2024*; IDC Quarterly Mobile Phone Tracker）。

つまり、物理的なインターネットアクセスは一定水準で確保されている。にもかかわらず、世界的にもっとも話題となっている技術トレンド（AI・LLM・生成AI）の認知がほぼゼロだった。

技術者の感覚としては、これはかなり異常な事態だ。L1（物理層）は機能しているのに、情報のスループットがゼロ。何が起きているのか、その構造を掘り下げたい。

## インターネット利用行動の非対称性

現地の事情に詳しい関係者から得た説明は、以下の通りだ。

**ザンビアの一般国民にとって、スマートフォンの主要用途はSNS（主にWhatsApp、Facebook）を介した知人・家族間のコミュニケーションである。**

情報検索（Google検索等）、ニュースサイトの閲覧、技術ブログの参照、動画コンテンツによる学習――こうしたインターネット利用パターンは一般的ではない。結果として、SNS上で流通するのは既知のコミュニティ内の情報に限られ、外部からの新規情報の流入が極めて限定的となっている。

これは、情報工学的に捉えると、ネットワークの「情報フロー」に関する問題だ。物理層（接続）は機能していても、アプリケーション層（利用行動）にボトルネックがあるために、情報のスループットが実質的にゼロに近くなっている。

## 技術普及経路の差異がリテラシ構造を規定する

この現象を理解するためには、日本（先進国）とザンビア（途上国）のインターネット技術の普及経路の差異を整理する必要がある。

**日本の技術普及経路：**
```
デスクトップPC → ダイヤルアップ/ADSL → ブロードバンド → Web検索・ニュース閲覧
→ SNS（mixi, Twitter等）登場 → スマートフォン普及
```

この経路では、ユーザーは「インターネット＝情報探索空間」という認知モデルを、PC時代に10年以上かけて形成している。SNSが登場した時点で、すでに情報検索・閲覧のリテラシが基盤として存在していた。スマートフォンは、PCで確立された利用パターンをモバイル化したものとして受容された。

**ザンビアの技術普及経路：**
```
（デスクトップPC・固定回線の段階をスキップ）
→ スマートフォン + モバイル回線（3G/4G）が同時に到達
→ この時点でSNS（WhatsApp, Facebook）がすでに普及済み
→ 初期接触アプリ＝SNS → スマートフォン＝交流ツールとして定着
```

この経路では、「ウェブブラウザでURLを入力して情報を取得する」「検索エンジンにクエリを投げて結果を評価する」といった基本的な情報リテラシが形成される機会がなかった。初めて触れたインターネット体験がSNSであり、それが支配的な利用パターンとして固定化された。

**同一のハードウェア（スマートフォン）に対し、技術普及の経路依存性（path dependency）が、まったく異なるユーザー行動モデルを生み出している。**

これは、技術移転やシステム導入の設計において無視できないファクターだ。

## 階層モデルで整理するデジタルデバイド

従来のデジタルデバイドの議論は、主にインフラ層に焦点を当てていた。「接続できるか/できないか」「デバイスがあるか/ないか」という二項対立だ。

しかし、ザンビアの事例は、デジタルデバイドを多層的に整理する必要性を示している。以下のような階層モデルで考えるとわかりやすい。

| 階層 | 内容 | ザンビアの状況 |
|------|------|----------------|
| L1: インフラ層 | ネットワーク接続、端末の所有 | 都市部は概ね充足 |
| L2: アクセス層 | アプリの利用、ウェブへの到達 | SNS利用はあるが、ブラウジング・検索は限定的 |
| L3: リテラシ層 | 情報の検索・取得・評価能力 | 検索行動の習慣が未形成 |
| L4: 活用層 | 取得した情報の業務・生活への適用 | 前提となるL3が不足しているため未到達 |

スマホ普及率のような統計は、L1の充足を示すものにすぎない。L2以降の階層は、統計では見えにくい。しかし、技術的なソリューションが現場で機能するためには、L4まで到達していなければならない。

ザンビアでは、L1は条件を満たしつつあるが、L2〜L3に大きな断絶がある。これが「スマホがあるのにAIを知らない」という一見矛盾した状況の技術的な説明だ。

## 水資源管理技術の導入における含意

本シリーズの主題であるザンビアの水資源管理について、このリテラシ構造がもたらす技術的な含意を整理する。

本プロジェクトでは、衛星降水量推定プロダクトと河川流量シミュレーション（物理モデル）を組み合わせた水資源把握の技術を導入しようとしている。具体的には以下の技術スタックが関わる。

- 衛星降水量推定（マイクロ波・赤外バンド複合プロダクト：JAXAのGSMaP（Global Satellite Mapping of Precipitation）やNASAのIMERG（Integrated Multi-satellitE Retrievals for GPM）が代表的なプロダクトであり、アフリカの水資源管理分野で広く利用されている）
- 分布型水文モデルによる河川流量シミュレーション
- インタラクティブなデータ可視化アプリケーション

前回の記事で述べた通り、ザンビアの水資源・土木分野では紙ベースのコミュニケーションが依然として主流であり、簡易的なプロトタイプの可視化アプリでも現場では非常に高い評価を受けた。日本では当然のように使われるダッシュボードや地図ベースの可視化が、ここでは初めて見る技術として受け取られる。

この状況を踏まえた技術導入の設計原則を、以下にまとめる。

### 1. UIの設計思想：情報密度を下げ、操作手順を最小化する

L3（リテラシ層）の制約を前提にUIを設計する。具体的には：
- 検索クエリの入力を必要としない設計（プリセット選択型）
- 階層的なメニュー構造ではなく、フラットなナビゲーション
- テキスト情報の密度を下げ、視覚的な表現（色、アイコン、マップ）を主体にする
- 操作から結果表示まで3タップ以内に収める

### 2. データの配信経路：既存の情報流通経路に乗せる

L2（アクセス層）の実態として、WhatsAppが支配的な通信チャネルであることを考慮すると、データや分析結果の配信にもこの経路を活用するアプローチが現実的だ。

- WhatsAppで送受信可能な形式（画像、短い動画、PDF）でのレポート自動生成
- WhatsApp Business APIやボットを活用した定期的なデータ配信（Meta社のBSP経由でザンビアでも利用可能。Twilio、360dialog等がサブサハラアフリカ地域をサポート）
- グループチャット経由での情報共有（現地の情報流通パターンに合致）

### 3. 技術のレイヤリング：段階的な複雑性の導入

フルスペックのGISダッシュボードをいきなり導入するのではなく、段階的に技術の複雑性を上げていく設計が現実的だ。

- Phase 1: 静的なマップ画像・PDFレポートの定期配信
- Phase 2: 簡易ウェブアプリ（閲覧のみ、入力不要）
- Phase 3: インタラクティブダッシュボード（条件選択・表示切替）
- Phase 4: ユーザーによるデータ入力・フィードバック機能

各フェーズの移行には、ユーザーのリテラシ層（L3）の発達状況を評価するメトリクスが必要だ。アクセスログ、利用頻度、機能利用率などを継続的にモニタリングし、次フェーズへの移行判断に用いる。

### 4. オフライン考慮：接続品質の不安定性への対応

モバイルネットワークの接続品質は地域や時間帯によって不安定であることを前提に、オフライン対応を織り込む。

- Progressive Web App（PWA）によるオフラインキャッシュ
- 軽量データ形式（GeoJSONの簡略化、画像の圧縮）の採用
- バックグラウンド同期による断続的接続への対応

## 見えなくなったデジタルデバイド

インターネットとグローバリゼーションが情報格差を大幅に縮小する――技術者としてそう信じていた。実際、インフラ層のデジタルデバイドは急速に縮小している。

しかし、ザンビアの現場が示したのは、L1（インフラ）の充足がL3（リテラシ）やL4（活用）の充足を自動的に導くわけではないという事実だ。技術普及の経路依存性が異なるユーザー行動モデルを生み、結果として「つながっているのに情報が流通しない」という新しい形のデジタルデバイドが発生している。

**旧来のデジタルデバイドは可視的だった（デバイスがない、回線がない）。新しいデジタルデバイドは不可視だ（デバイスはある、回線もある、しかし情報は流れていない）。**

統計で見えるL1の改善をもって「デジタルデバイドは解消された」と判断することは、危険な認識の誤りだ。技術を現場に届ける立場にある者として、L2〜L4の階層を見据えた設計が求められる。

水資源管理に限らず、途上国向けの技術移転プロジェクト全般において、この多層構造を認識し、各階層に応じた対策を講じることが、技術が実際に「使われる」ための前提条件となる。

繰り返すが、**L1の充足はL3・L4の充足を保証しない。** 「インフラが整えばあとは自然に使われる」という思い込みは、技術者にとって最も危険なバイアスだ。

スマホの画面は同じだ。でも、その画面の向こうに見えている世界の広さは、技術普及の歴史によってまったく異なる。その非対称性を、技術の力で橋渡しすることが、我々の仕事だ。
